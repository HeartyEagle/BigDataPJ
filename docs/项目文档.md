# 项目文档

## 1. 项目背景

本项目面向运维监控场景，对节点、容器、JVM、Istio、业务黄金指标等时间序列进行异常检测。目标是处理 5000+ 条时间序列、3GB+ 数据规模，并在三台服务器 Hadoop 集群环境中展示清洗、检测、并行运行和可视化结果。

## 2. 数据说明

性能指标统一字段：

```text
timestamp, cmdb_id, kpi_name, value
```

业务指标统一字段：

```text
timestamp, service, rr, sr, count, mrt
```

一条时间序列定义为：

```text
cmdb_id + kpi_name
```

## 3. 系统架构

```text
原始数据
  -> 数据导入
  -> 数据清洗
  -> 上传 HDFS
  -> 序列构造和统计
  -> Hadoop Streaming / MapReduce 异常检测
  -> HDFS 结果汇总
  -> Demo 可视化
```

Hadoop MapReduce 设计：

```text
Mapper:
  输入 timestamp, cmdb_id, kpi_name, value
  输出 key = cmdb_id|kpi_name, value = timestamp,value

Reducer:
  每个 key 对应一条时间序列
  对该序列计算 IQR/K-Sigma/滑动窗口等异常检测结果
  输出统一异常结果字段
```

## 4. 算法设计

Baseline 算法：

```text
IQR
K-Sigma
Range
```

高级算法方向：

```text
滑动窗口 K-Sigma 或 LOF
```

## 5. 集群设计

```text
服务器 1：NameNode / ResourceManager / 结果汇总 / Demo
服务器 2：DataNode / NodeManager / MapReduce Worker
服务器 3：DataNode / NodeManager / MapReduce Worker / 性能测试
```

## 6. 输出结果

```text
results/profiles/       数据统计
results/anomalies/      异常结果
results/performance/    性能测试
```
