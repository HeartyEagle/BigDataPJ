## 2. Advanced 算法说明

本项目的高级算法实现为 **Rolling Robust Z-Score**，对应代码：

```text
src/algorithms/advanced.py
src/hadoop/reducer_advanced.py
```

算法以 `cmdb_id + kpi_name` 作为一条时间序列，对每条序列按时间排序后独立检测异常。

### 2.1 核心思想

传统全局 IQR 或 K-Sigma 使用整条序列的固定阈值，适合分布稳定的指标；但系统监控数据常出现局部波动、趋势变化和阶段性负载变化。Rolling Robust Z-Score 使用历史滑动窗口估计当前点的局部中心和局部波动范围，因此更适合捕捉局部异常。

对每个时间点 `x_t`：

1. 取当前点之前的历史窗口，默认窗口大小 `window=60`。
2. 使用历史窗口的 median 作为局部中心。
3. 使用历史窗口的 IQR 估计 robust scale：

```text
scale = (Q3 - Q1) / 1.349
```

4. 计算 robust z-score：

```text
score = abs(x_t - center) / scale
```

5. 默认阈值 `z_threshold=3.5`。若 `score > 3.5`，则标记为异常。

对应输出阈值：

```text
threshold_low  = center - 3.5 * scale
threshold_high = center + 3.5 * scale
```

### 2.2 冷启动与鲁棒性处理

对于序列开头历史点不足的情况，算法使用全局 fallback：

```text
global_center = median(values)
global_scale  = max(global_iqr_scale, global_std)
```

其中：

```text
global_iqr_scale = (Q3(values) - Q1(values)) / 1.349
```

这样可以避免序列开头没有足够历史窗口时阈值不可用。

算法使用 `history = values.shift(1)`，即当前点只使用它之前的历史数据估计阈值，避免当前异常点污染自己的检测阈值。

### 2.3 相比 baseline 的特点

| 方法 | 阈值类型 | 适用场景 | 局限 |
| --- | --- | --- | --- |
| IQR | 全局四分位阈值 | 分布稳定、有离群点的数据 | 对趋势变化不敏感 |
| K-Sigma | 全局均值和标准差 | 近似正态分布的数据 | 容易受极端值影响 |
| Range | 规则阈值 | 明显非法值，如负值、不合理百分比 | 只能发现规则异常 |
| Rolling Robust Z-Score | 局部滑动窗口 median + IQR | 局部突增、突降、趋势变化中的异常 | 计算成本高于全局方法 |

### 3.2 加速比

加速比计算方式：

```text
speedup = local_runtime / hadoop_runtime
```

| 方法 | Local 时间(s) | Hadoop 时间(s) | Hadoop 加速比 |
| --- | ---: | ---: | ---: |
| IQR | 444 | 176 | 2.52x |
| K-Sigma | 547 | 174 | 3.14x |
| Range | 397 | 171 | 2.32x |
| Advanced | 565 | 193 | 2.93x |

从结果看，四种算法在 Hadoop 上均获得明显加速。其中 K-Sigma 加速比最高，为 3.14x；Advanced 高级算法在 Hadoop 上也达到 2.93x 加速。

### 3.3 异常比例

| 方法 | 异常数 | 异常比例 |
| --- | ---: | ---: |
| IQR | 389,827 | 1.6018% |
| K-Sigma | 65,665 | 0.2698% |
| Range | 51,840 | 0.2130% |
| Advanced | 330,333 | 1.3573% |

Range 的 Hadoop 结果已按修复后的阈值规则重新分析，异常点数为 51,840，异常率为 0.2130%，主要集中在 JVM Metaspace / NonHeap 内存指标的非法负值。旧 Local Range 异常数 1,153,040 是修复前结果，保留在性能表中仅用于说明当时性能实验的输入输出规模，不建议作为 PPT 当前结论。IQR 和 Advanced 更适合展示典型监控波动异常，K-Sigma 在统计类算法中最保守。
