# Master 环境部署说明（有外网版）

本文档用于在 `master/Hadoop01` 上配置项目环境。worker 节点只需要 Java、Hadoop、Python 基础环境；数据清洗、Demo 和结果汇总优先放在 master 上执行。

## 1. 推荐目录

Linux 本地目录：

```text
/opt/bigdatapj/
├── app/                 # 项目代码
├── software/            # 安装包或临时下载文件
├── data/
│   ├── raw/             # 原始数据备份
│   ├── cleaned/         # 清洗结果备份
│   └── sample/          # 小样本
├── logs/
└── outputs/
```

HDFS 目录：

```text
/bigdatapj/
├── data/
│   ├── raw/
│   └── cleaned/
├── results/
│   ├── profiles/
│   ├── anomalies/
│   └── performance/
└── tmp/
```

创建本地目录：

```bash
mkdir -p /opt/bigdatapj/{app,software,data/raw,data/cleaned,data/sample,logs,outputs}
```

## 2. 安装系统基础软件

CentOS 7 如果 yum 源可用，先安装基础工具：

```bash
yum install -y vim wget curl net-tools openssh-clients rsync tar unzip git
```

如果 `yum` 源不可用，先把 CentOS 7 源切到可访问的镜像源，再执行上面命令。

## 3. 安装 Java

Hadoop 推荐 Java 8：

```bash
yum install -y java-1.8.0-openjdk java-1.8.0-openjdk-devel
java -version
```

配置环境变量：

```bash
cat >> ~/.bashrc <<'EOF'

# Java
export JAVA_HOME=/usr/lib/jvm/java-1.8.0-openjdk
export PATH=$JAVA_HOME/bin:$PATH
EOF

source ~/.bashrc
```

成功标志：

```bash
java -version
```

能看到 `1.8.0`。

## 4. 安装 Python

CentOS 7 自带 Python 版本通常太老，例如：

```bash
python3 --version
```

如果看到 `Python 3.6.x`，不要直接安装 `requirements.txt`。`pandas>=2.0` 需要 Python 3.8 以上，系统 Python 3.6 只能安装到很老的 pandas。

推荐在 master 上安装 Miniconda，并创建 Python 3.12 项目环境：

```bash
cd /opt/bigdatapj/software
wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh
bash Miniconda3-latest-Linux-x86_64.sh
```

安装路径建议填写：

```text
/opt/miniconda3
```

安装完成后重新加载环境：

```bash
source ~/.bashrc
conda --version
```

创建项目环境：

```bash
conda create -n bigdatapj python=3.12 -y
conda activate bigdatapj
```

成功标志：

```bash
python --version
```

应显示 `Python 3.12.x`。

### 4.1 CentOS 7 证书问题

如果 `wget` 报证书错误，例如：

```text
Unable to locally verify the issuer's authority
```

先尝试更新 CA 证书：

```bash
yum install -y ca-certificates
update-ca-trust force-enable
update-ca-trust extract
```

然后重新下载 Miniconda。

如果仍然失败，临时跳过证书校验：

```bash
cd /opt/bigdatapj/software
wget --no-check-certificate https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh
```

下载后先确认文件存在：

```bash
ls -lh Miniconda3-latest-Linux-x86_64.sh
```

看到几十 MB 的安装包后再执行：

```bash
bash Miniconda3-latest-Linux-x86_64.sh
```

如果 `conda create` 也因为 SSL 报错，可以临时关闭 conda 的 SSL 校验：

```bash
conda config --set ssl_verify false
conda create -n bigdatapj python=3.12 -y
conda config --set ssl_verify true
```

### 4.2 安装项目 Python 依赖

```bash
conda activate bigdatapj
cd /opt/bigdatapj/app
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

成功标志：

```bash
python --version
python -c "import pandas, numpy, streamlit, plotly, sklearn; print('python env ok')"
```

后续在 master 上运行项目，都先执行：

```bash
conda activate bigdatapj
cd /opt/bigdatapj/app
```

## 5. Python 依赖说明

依赖统一写在项目根目录：

```text
requirements.txt
```

当前包含：

```text
pandas          数据清洗、统计、CSV/Parquet 读取
numpy           baseline 算法计算
pyarrow         Parquet 文件支持
streamlit       Demo 页面
plotly          Demo 图表
PyYAML          读取 configs/pipeline.yaml
openpyxl        读取训练数据说明.xlsx 或 Excel 数据说明
scikit-learn    后续 LOF 等高级算法
```

注意：`src/hadoop/mapper_baseline.py` 和 `src/hadoop/reducer_baseline.py` 尽量只使用 Python 标准库；`src/hadoop/reducer_advanced.py` 会使用 pandas/numpy，因此 Hadoop Advanced 必须依赖三台机器上同路径的 conda 环境。Hadoop Streaming 脚本里如果要指定解释器，优先使用：

```text
/opt/miniconda3/envs/bigdatapj/bin/python
```

如果 worker 节点不能联网、也没有安装 Miniconda，可以在 master 上使用 `conda-pack` 打包并通过 SSH 流式解压到相同路径：

```bash
conda install -n base -y conda-pack
conda-pack -n bigdatapj -o /tmp/bigdatapj-py312.tar.gz

cat /tmp/bigdatapj-py312.tar.gz | ssh root@10.176.62.222 \
  'rm -rf /opt/miniconda3/envs/bigdatapj && mkdir -p /opt/miniconda3/envs/bigdatapj && tar -xzf - -C /opt/miniconda3/envs/bigdatapj && /opt/miniconda3/envs/bigdatapj/bin/conda-unpack'

cat /tmp/bigdatapj-py312.tar.gz | ssh root@10.176.62.223 \
  'rm -rf /opt/miniconda3/envs/bigdatapj && mkdir -p /opt/miniconda3/envs/bigdatapj && tar -xzf - -C /opt/miniconda3/envs/bigdatapj && /opt/miniconda3/envs/bigdatapj/bin/conda-unpack'
```

两台 worker 不需要联网，也不需要预先安装 Miniconda；只要最终路径存在并能执行下面命令即可：

```bash
/opt/miniconda3/envs/bigdatapj/bin/python -c "import pandas, numpy, sklearn; print('worker python env ok')"
```

## 6. 安装 Hadoop

下载 Hadoop：

```bash
cd /opt/bigdatapj/software
wget https://archive.apache.org/dist/hadoop/common/hadoop-3.3.6/hadoop-3.3.6.tar.gz
tar -zxf hadoop-3.3.6.tar.gz -C /opt
ln -s /opt/hadoop-3.3.6 /opt/hadoop
```

配置环境变量：

```bash
cat >> ~/.bashrc <<'EOF'

# Hadoop
export HADOOP_HOME=/opt/hadoop
export HADOOP_CONF_DIR=$HADOOP_HOME/etc/hadoop
export PATH=$HADOOP_HOME/bin:$HADOOP_HOME/sbin:$PATH
EOF

source ~/.bashrc
```

成功标志：

```bash
hadoop version
```

能看到 Hadoop 版本信息。

## 7. 配置项目环境变量

项目代码统一放在 `/opt/bigdatapj/app`，也就是 `app` 目录本身就是 Git 仓库根目录：

```bash
cat >> ~/.bashrc <<'EOF'

# BigDataPJ
export BIGDATAPJ_HOME=/opt/bigdatapj/app
export PYTHONPATH=$BIGDATAPJ_HOME:$PYTHONPATH
EOF

source ~/.bashrc
cd $BIGDATAPJ_HOME
```

成功标志：

```bash
python -m compileall src demo
```

没有报错即可。

## 11. 项目运行成功标志

清洗成功：

```bash
python -m src.preprocessing.clean_metrics --input data/raw/metrics_raw.csv --output data/cleaned/metrics_cleaned.csv
```

能生成：

```text
data/cleaned/metrics_cleaned.csv
```

样本生成成功：

```bash
python -m src.preprocessing.make_samples --input data/cleaned/metrics_cleaned.csv --chunksize 500000
```

能生成：

```text
data/sample/sample_small.csv
data/sample/sample_medium.csv
```

HDFS 全量 cleaned 数据按 Hadoop 输出目录交接，当前约定目录为：

```text
/bigdatapj/data/cleaned
```

该目录应直接包含 `_SUCCESS` 和 `part-*` 文件，例如：

```bash
hdfs dfs -ls /bigdatapj/data/cleaned
```

能看到：

```text
_SUCCESS
part-00000
part-00001
```

如需把全量 HDFS cleaned 合并为本地单 CSV，用于单机 baseline 或 Hadoop 性能对比：

```bash
mkdir -p data/cleaned
{ printf "timestamp,cmdb_id,kpi_name,value\n"; hdfs dfs -cat /bigdatapj/data/cleaned/part-* | sed 's/\t$//'; } > data/cleaned/metrics_cleaned.csv
```

当前本地合并文件为：

```text
data/cleaned/metrics_cleaned.csv
```

Hadoop 作业成功：

```bash
hdfs dfs -ls /bigdatapj/results/anomalies/hadoop_iqr
```

能看到：

```text
_SUCCESS
part-00000
part-00001
```

查看结果：

```bash
hdfs dfs -cat /bigdatapj/results/anomalies/hadoop_iqr/part-* | head
```

能看到字段：

