# 时间序列异常检测项目组长交付说明

本文档是组长给全组的开发入口。所有人按这里的接口交付，最后才能顺利集成。

## 1. 总目标

把监控指标数据处理成 5000+ 条时间序列，使用 Hadoop/HDFS/MapReduce 完成基础算法和高级/并行算法异常检测，并用 Demo 展示数据规模、异常点、算法对比和性能对比。

## 2. 数据接口

性能指标来自 `训练数据说明.xlsx` 的“性能指标-数据格式”：

```text
timestamp, cmdb_id, kpi_name, value
```

字段含义：

| 字段 | 含义 |
| --- | --- |
| timestamp | 10 位秒级时间戳，清洗后转成可读时间 |
| cmdb_id | 指标所在对象，例如 node/container/jvm/istio 对象 |
| kpi_name | 指标名，例如 system.cpu.pct_usage |
| value | 指标值，必须转成数值 |

业务指标来自“业务指标-数据格式”，黄金指标包括：

```text
timestamp, service, rr, sr, count, mrt
```

本项目第一阶段优先跑通性能指标；业务指标可作为 Demo 或扩展算法输入。

## 8. 成员 5：高级算法 / 并行计算 / 性能测试

需要修改或新增：

```text
src/distributed/shard_runner.py
src/hadoop/mapper_baseline.py
src/hadoop/reducer_baseline.py
src/hadoop/reducer_advanced.py
scripts/hadoop/run_streaming_baseline.sh
scripts/hadoop/run_streaming_advanced.sh
src/algorithms/advanced.py
results/anomalies/local_advanced_full/anomaly_advanced.csv
results/anomalies/local_advanced_full/advanced_summary.csv
results/anomalies/local_advanced_full/demo_cases.csv
```

建议路线：

```text
先跑通 Hadoop Streaming IQR，默认输入为 `/bigdatapj/data/cleaned`
再扩展高级算法；当前本地版为 Rolling-Robust-ZScore，使用滑动窗口 median + IQR 局部阈值
最后用 `data/cleaned/metrics_cleaned.csv` 做本地单机 vs Hadoop 双 Worker 性能对比
```

Mapper 可以复用，算法差异放在 Reducer：

```text
Hadoop Baseline = mapper_baseline.py + reducer_baseline.py --method iqr|ksigma|range
Hadoop Advanced = mapper_baseline.py + reducer_advanced.py
```

当前本地高级算法已在 `data/sample/sample_medium.csv` 跑通，输出：

```text
results/anomalies/local_advanced_full/anomaly_advanced.csv
results/anomalies/local_advanced_full/advanced_summary.csv
results/anomalies/local_advanced_full/demo_cases.csv
```

性能结果格式：

```text
method, server_num, series_count, data_count, runtime_sec, throughput
```

完整全量对比可直接运行：

```bash
bash scripts/run_all/run_full_comparison.sh
```

汇总输出：

```text
results/performance/full_comparison_report.csv
```

第一步先跑通最小闭环：

```text

```

第二步扩展：

```text

补齐 K-Sigma 和 Range

```

第三步加分：

```text

Hadoop MapReduce 并行
性能测试
PPT 和完整项目文档
```
