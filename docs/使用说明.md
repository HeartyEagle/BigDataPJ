# 使用说明

## 1. 安装依赖

master 节点推荐使用 Miniconda 环境。CentOS 7 自带 Python 3.6 太老，不要直接安装本项目依赖。

```bash
cd /opt/bigdatapj/app
conda activate bigdatapj
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Windows 本地调试也可以直接执行：

```powershell
python -m pip install -r requirements.txt
```

完整服务器部署步骤见 `docs/环境部署.md`。

## 2. 准备数据

成员 2 将导入后的性能指标数据放到：

```text
data/raw/metrics_raw.csv
```

字段必须包含：

```text
timestamp, cmdb_id, kpi_name, value
```

## 3. 运行主流程

本地调试流程：

```bash
bash scripts/run_all/run_pipeline.sh --input-path data/raw/metrics_raw.csv
```

Hadoop 集群流程。全量 cleaned 数据已经按 Hadoop 目录形式放在 HDFS：

```text
/bigdatapj/data/cleaned
```

该目录内包含 `_SUCCESS` 和多个 `part-*` 文件，后续 Hadoop 作业直接把这个目录作为输入。

```bash
bash scripts/hadoop/run_streaming_baseline.sh \
  --input /bigdatapj/data/cleaned \
  --output /bigdatapj/results/anomalies/hadoop_iqr
```

同时，为了让单机 baseline 和性能对比可以读取完整数据，已将 HDFS cleaned 合并为本地 CSV：

```text
data/cleaned/metrics_cleaned.csv
```

如需重新生成：

```bash
mkdir -p data/cleaned
{ printf "timestamp,cmdb_id,kpi_name,value\n"; hdfs dfs -cat /bigdatapj/data/cleaned/part-* | sed 's/\t$//'; } > data/cleaned/metrics_cleaned.csv
```

调试样本由 `make_samples.py` 分块扫描生成，适合当前 1.9G 全量 CSV：

```bash
python -m src.preprocessing.make_samples --input data/cleaned/metrics_cleaned.csv --chunksize 500000
```

当前样本文件：

```text
data/sample/sample_small.csv   10 series, 27060 rows
data/sample/sample_medium.csv  100 series, 280811 rows
```

流程会依次生成：

```text
data/cleaned/metrics_cleaned.csv
data/sample/sample_small.csv
data/sample/sample_medium.csv
results/profiles/*.csv / *.json
results/anomalies/anomaly_iqr.csv
results/anomalies/anomaly_ksigma.csv
results/anomalies/anomaly_range.csv
results/anomalies/baseline_summary.csv
results/anomalies/demo_cases.csv
```

成员 5 本地高级算法：

```bash
python -m src.algorithms.advanced \
  --input data/sample/sample_medium.csv \
  --output-dir results/anomalies/local_advanced_full
```

Advanced 输出目录内包含：

```text
anomaly_advanced.csv
advanced_summary.csv
demo_cases.csv
```

该算法为滚动 Robust Z-Score：对每条 `cmdb_id + kpi_name` 序列，使用历史滑动窗口 median + IQR 估计局部中心和波动范围。当前已在 `sample_medium.csv` 上跑通：

```text
series=100
rows=280811
anomalies=3873
```

Hadoop 结果会先输出到：

```text
/bigdatapj/results/anomalies/hadoop_iqr
```

Hadoop Advanced 使用同一个 mapper，换成高级算法 reducer：

```bash
bash scripts/hadoop/run_streaming_advanced.sh \
  --input /bigdatapj/data/cleaned \
  --output /bigdatapj/results/anomalies/hadoop_advanced
```

两组全量性能对比应使用同一份 cleaned 数据：

```text
Local IQR       data/cleaned/metrics_cleaned.csv
Hadoop-IQR      /bigdatapj/data/cleaned
Local Advanced  data/cleaned/metrics_cleaned.csv
Hadoop Advanced /bigdatapj/data/cleaned
```

Hadoop baseline reducer 支持 `iqr`、`ksigma`、`range`，与本地 baseline 对齐：

```bash
bash scripts/hadoop/run_streaming_baseline.sh \
  --input /bigdatapj/data/cleaned \
  --output /bigdatapj/results/anomalies/hadoop_ksigma \
  --method ksigma
```

一键跑完整全量对比：

```bash
bash scripts/run_all/run_full_comparison.sh
```

脚本会依次运行 Local IQR/K-Sigma/Range、Hadoop IQR/K-Sigma/Range、Local Advanced、Hadoop Advanced，并输出：

```text
results/performance/full_comparison_report.csv
```

如果需要额外文本日志，可加 `--time-log results/performance/full_comparison_summary.txt`。

再下载到：

```text
results/anomalies/hadoop_iqr/
```

## 4. 启动 Demo

```bash
streamlit run demo/app.py
```

## 5. 常见问题

如果清洗阶段报缺字段，先检查原始数据是否包含：

```text
timestamp, cmdb_id, kpi_name, value
```

如果 Demo 没有图，先确认是否已经生成 `results/profiles/` 和 `results/anomalies/`。
