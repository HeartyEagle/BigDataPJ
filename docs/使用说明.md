# 使用说明

## 1. 安装依赖

master 节点推荐使用 Miniconda 环境。CentOS 7 自带 Python 3.6 太老，不要直接安装本项目依赖。

```bash
cd /opt/bigdatapj/app
conda activate bigdatapj
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Windows 本地调试也可以直接执行：

```powershell
python -m pip install -r requirements.txt
```

完整服务器部署步骤见 `docs/环境部署.md`。

## 2. 准备数据

成员 2 将导入后的性能指标数据放到：

```text
data/raw/metrics_raw.csv
```

字段必须包含：

```text
timestamp, cmdb_id, kpi_name, value
```

## 3. 运行主流程

本地调试流程：

```powershell
.\scripts\run_all\run_pipeline.ps1 -InputPath data/raw/metrics_raw.csv
```

Hadoop 集群流程：

```powershell
.\scripts\run_all\run_pipeline.ps1 -InputPath data/raw/metrics_raw.csv -RunHadoop
```

流程会依次生成：

```text
data/cleaned/metrics_cleaned.csv
results/profiles/*.csv / *.json
results/anomalies/anomaly_iqr.csv
results/anomalies/anomaly_ksigma.csv
results/anomalies/anomaly_range.csv
```

Hadoop 结果会先输出到：

```text
/bigdatapj/results/anomalies/hadoop_iqr
```

再下载到：

```text
results/anomalies/hadoop_iqr/
```

## 4. 启动 Demo

```powershell
streamlit run demo/app.py
```

## 5. 常见问题

如果清洗阶段报缺字段，先检查原始数据是否包含：

```text
timestamp, cmdb_id, kpi_name, value
```

如果 Demo 没有图，先确认是否已经生成 `results/profiles/` 和 `results/anomalies/`。
